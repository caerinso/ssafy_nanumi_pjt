# 브랜치 전략

-   main: 출시 전 까지 작업 X
    -   dev : 분야별 기능 합쳐서 테스트
        -   backend : BE 관련 완료된 기능 
        -   frontend : FE 관련 완료된 기능
        
# Commit Message Convention

YYMMDD_AI_AUTHOR_TYPE_CONTENT

230207_BE_CHEONG_INIT_AddUserRepository

## TYPE

-   INIT : 신규 파일 생성
-   FEAT : 신규 기능 생성
-   MODI : 기능 수정
-   FIX : 버그 수정
-   TEST : 테스트
